# RegenTwin: Мультимасштабная платформа моделирования регенерации тканей

## Отчёт о текущем функционале проекта

---

## 1. Введение и постановка задачи

Проект **RegenTwin** — это вычислительная платформа для моделирования процессов регенерации биологических тканей с учётом терапевтических воздействий (PRP-терапия и PEMF-терапия). Платформа объединяет три уровня моделирования — макроскопический (система стохастических дифференциальных уравнений), микроскопический (агентная модель) и их двунаправленную интеграцию.

**Цель проекта** — создать цифровой двойник (digital twin) процесса заживления раны, который:
- моделирует динамику 20 биологических переменных во времени;
- учитывает стохастическую природу биологических процессов;
- интегрирует данные проточной цитометрии (FCS-файлы) для персонализации параметров;
- позволяет сравнивать эффективность терапевтических протоколов;
- предоставляет REST API и веб-интерфейс для интерактивной работы.

## 2. Математическое ядро

### 2.1. Расширенная система СДУ (Extended SDE) — 20 переменных

Основу модели составляет система из **20 стохастических дифференциальных уравнений**, описывающих динамику заживления раны. Переменные разделены на четыре группы:

#### Клеточные популяции (8 переменных)

| Переменная | Биологический смысл | Ед. измерения |
|:----------:|:------------------:|:-------------:|
| P | Тромбоциты | клетки/мкл |
| Ne | Нейтрофилы | клетки/мкл |
| M1 | Макрофаги M1 (провоспалительные) | клетки/мкл |
| M2 | Макрофаги M2 (репаративные) | клетки/мкл |
| F | Фибробласты | клетки/мкл |
| Mf | Миофибробласты (α-SMA+) | клетки/мкл |
| E | Эндотелиальные клетки (CD31+) | клетки/мкл |
| S | Стволовые клетки (CD34+) | клетки/мкл |

#### Цитокины (7 переменных)

| Переменная | Биологический смысл | Роль |
|:----------:|:------------------:|:----:|
| C_TNF | TNF-α | Провоспалительный |
| C_IL10 | IL-10 | Противовоспалительный |
| C_PDGF | PDGF | Фактор роста тромбоцитов |
| C_VEGF | VEGF | Фактор роста эндотелия |
| C_TGFβ | TGF-β | Трансформирующий фактор (ключевой для бистабильности фиброза) |
| C_MCP1 | MCP-1 | Хемоаттрактант моноцитов |
| C_IL8 | IL-8 | Хемокин нейтрофилов |

#### Компоненты внеклеточного матрикса (3 переменных)

| Переменная | Биологический смысл |
|:----------:|:------------------:|
| ρ_collagen | Плотность коллагена (нормализованная, 0–1) |
| C_MMP | Матриксные металлопротеиназы |
| ρ_fibrin | Плотность фибрина (временный каркас) |

#### Вспомогательные переменные (2)

| Переменная | Биологический смысл |
|:----------:|:------------------:|
| D | DAMPs — сигнал повреждения (экспоненциальный распад, τ = 36 ч) |
| O₂ | Парциальное давление кислорода (ммHg) |

#### Ключевые уравнения

Каждая переменная описывается уравнением вида:

```
dX = f(X, t) dt + σ(X) dW
```

где f — функция дрейфа, σ — шумовой коэффициент, W — винеровский процесс.

**Примеры функций дрейфа:**

Поляризация макрофагов M1 ↔ M2:
```
dM1/dt = R_M · φ₁ - k_switch · ψ(IL10, TGFβ) · M1 + k_reverse · ζ(TNF) · M2 - δ_M · M1
dM2/dt = R_M · φ₂ + k_switch · ψ · M1 - k_reverse · ζ · M2 - δ_M · M2
```
где φ₁ = TNF / (TNF + IL10 + ε) — функция поляризации, ψ — функция переключения (Hill-функция от IL-10 + TGF-β).

Бистабильность фиброза (TGF-β → Mf → TGF-β):
```
dMf/dt = k_act · F · A(TGFβ) - δ_Mf · Mf · (1 - TGFβ / (K_survival + TGFβ))
dC_TGFβ/dt = s_P · k_deg · P + s_M2 · M2 + s_Mf · Mf + Θ_PRP_TGF - γ_TGF · C_TGFβ
```
Здесь миофибробласты (Mf) производят TGF-β, который в свою очередь активирует фибробласты в миофибробласты — это реализует **положительную обратную связь**, приводящую к бистабильному поведению (нормальное заживление vs. фиброз).

Гипоксия-зависимый ангиогенез:
```
dE/dt = r_E · E · (1 - E/K_E) · V(VEGF) · (1 - θ(O₂)) - δ_E · E
```
где θ(O₂) = O₂/(K_O2 + O₂) — фактор гипоксии. При низком O₂ усиливается пролиферация эндотелия.

**Численное интегрирование** выполняется методом **Эйлера-Маруямы** с мультипликативным шумом σ(X) = σ · X (геометрическое броуновское движение). Набор параметров включает **80+ биофизических констант**.

### 2.2. Упрощённая модель СДУ (MVP SDE) — 2 переменные

Для быстрого прототипирования и валидации реализована упрощённая 2-переменная модель:
- N — плотность клеток (cells/cm²)
- C — концентрация цитокинов (ng/mL)

С логистическим ростом и линейной деградацией, модулируемой терапией.

### 2.3. Агентная модель (ABM)

Агентная модель описывает **микроскопическую** динамику на пространственной сетке 100×100 мкм.

**6 типов агентов:**

| Тип агента | Маркер | Начальное количество |
|:----------:|:------:|:-------------------:|
| Стволовая клетка | CD34+ | 50 |
| Макрофаг | CD14+/CD68+ | 30 |
| Фибробласт | — | 20 |
| Нейтрофил | CD66b+ | 0 (рекрутируются) |
| Эндотелиальная клетка | CD31+ | 10 |
| Миофибробласт | α-SMA+ | 0 (дифференцируются) |

**Правила взаимодействия:**
- Случайное блуждание с хемотаксисом (коэффициент диффузии 1.0 мкм²/ч)
- Контактное ингибирование: деление блокируется при >5 соседях в радиусе 2 мкм
- Эффероцитоз: макрофаги поглощают нейтрофилы → продукция IL-10
- Поляризация M1 → M2: управляется TGF-β и IL-10
- Активация фибробласт → миофибробласт: TGF-β зависимая (путь Smad2/3)
- Ангиогенез: VEGF-зависимая пролиферация эндотелия (усиливается при гипоксии)
- Максимум агентов: 10 000

Граничные условия: периодические, отражающие или поглощающие (конфигурируемые). Пространственный индекс: хэш-сетка или KD-дерево.

### 2.4. Интеграция SDE–ABM

Связывание двух масштабов реализовано методом **двунаправленного операторного расщепления (bidirectional operator splitting)**:

**SDE → ABM:**
- Глобальное состояние SDE пространственно распределяется среди агентов ABM
- Цитокиновые поля (TNF-α, IL-8, PDGF, VEGF, TGF-β) вычисляются на сетке 10 мкм
- Каждый агент воспринимает локальный градиент концентрации

**ABM → SDE:**
- Плотность агентов агрегируется в макроскопическую переменную N
- Суммарная секреция агентов → макроскопическая продукция C
- Коррекция расхождений: |SDE_N − ABM_total| → correction_rate × discrepancy

Интервал синхронизации: 1 час. Сила связи: 0.5. Порог максимального расхождения: 50%.

## 3. Модели терапевтических воздействий

### 3.1. PRP-терапия (Platelet-Rich Plasma)

Модель PRP реализует **бифазную кинетику высвобождения** факторов роста:

```
Θ_PRP_i(t) = dose · c0_i · [exp(-t/τ_burst) - exp(-t/τ_sustained)] / (τ_burst - τ_sustained) · exp(-2t/τ_sustained)
```

**4 фактора роста с индивидуальной кинетикой:**

| Фактор | c₀ (нг/мл) | τ_burst (ч) | τ_sustained (ч) |
|:------:|:----------:|:-----------:|:---------------:|
| PDGF | 20 | 1 | 48 |
| VEGF | 1.0 | 1 | 24 |
| TGF-β | 30 | 2 | 72 |
| EGF | 0.2 | 0.5 | 12 |

Доза PRP: 4× (концентрация в 3–5 раз выше цельной крови). Рекрутинг стволовых клеток: α_PRP_S = 0.5 (через путь SDF-1/CXCR4).

### 3.2. PEMF-терапия (Pulsed Electromagnetic Field)

Модель PEMF реализует **3 биофизических механизма:**

1. **Противовоспалительный (аденозиновый путь A₂A/A₃):**
   - Снижение TNF-α на 30–50%
   - Оптимальная частота: 27.12 Гц
   - ε_max = 0.4

2. **Пролиферативный (Ca²⁺-CaM/NO путь):**
   - Стимуляция пролиферации фибробластов и эндотелия
   - Оптимальная частота: 75 Гц
   - ε_max = 0.3

3. **Миграционный (MAPK/ERK путь):**
   - Увеличение подвижности клеток в ABM
   - ε_max = 0.25

### 3.3. Синергия PRP + PEMF

```
synergy(t) = 1 + β_synergy · Θ_PRP(t) · PEMF_active(t)
```
β_synergy = 0.2 — суперадитивный эффект через Ca²⁺-рецепторную сенситизацию.

## 4. Детектор фаз заживления раны

Реализован **4-фазный фреймворк** определения текущей фазы раневого процесса:

| Фаза | Типичные сроки | Ключевые маркеры |
|:----:|:--------------:|:----------------:|
| **Гемостаз** | 0–6 ч | ↑P, ↑фибрин, ↑DAMPs, ↓иммунные клетки |
| **Воспаление** | 6 ч – 4–6 дн | ↑Ne, M1 > M2, ↑TNF-α, ↑IL-8 |
| **Пролиферация** | 4–21 дн | ↑F, M2 > M1, ↑коллаген, ↑VEGF, ↑E |
| **Ремоделирование** | 21 дн – 1 год | стабильный коллаген, активный MMP/TIMP баланс, ↓Ne, ↓M1, ↓Mf |

Каждая фаза определяется взвешенной комбинацией биомаркеров с вычислением **степени уверенности** (confidence). Метод `WoundPhaseDetector.detect_phase(state)` возвращает текущую фазу, уверенность, доминирующие клетки и цитокины.

## 5. Метод Монте-Карло

Для учёта стохастической природы модели реализованы **ансамблевые симуляции Монте-Карло**:

- Число реализаций: 100–1000 (конфигурируемо)
- Параллельное выполнение через ThreadPoolExecutor
- Квантили: 5%, 25%, 50% (медиана), 75%, 95%
- Статистики: среднее, стандартное отклонение, доверительные интервалы
- Отслеживание успешных/неуспешных траекторий

Дополнительные возможности:
- **Sweep параметров** — варьирование одного параметра с MC-ансамблем для каждого значения
- **Сравнение терапий** — бенчмарк PRP vs PEMF vs комбинированная vs контроль

## 6. Интеграция данных проточной цитометрии

### 6.1. Парсинг FCS-файлов

Модуль `FCSLoader` поддерживает форматы FCS 2.0, 3.0, 3.1 (через библиотеку FlowKit):
- Загрузка с опциональным субсэмплингом
- Извлечение каналов, событий (сырые/компенсированные/трансформированные), метаданных
- Валидация наличия необходимых каналов (нечувствительно к регистру)
- Конвертация в pandas DataFrame

### 6.2. Извлечение параметров модели

Данные проточной цитометрии автоматически трансформируются в начальные условия модели:

```
FCS-файл → Гейтинг → ModelParameters:
  - n0 (плотность клеток из гейтинга)
  - stem_cell_fraction (%CD34+ / общее)
  - macrophage_fraction (%(CD14+ или CD68+) / общее)
  - apoptotic_fraction (%Annexin+ / общее)
  - c0 (концентрация факторов роста)
  - inflammation_level (относительный уровень TNF-α, 0–1)
```

Для расширенной модели доступен `ExtendedModelParameters` — полный набор из 20 начальных условий.

## 8. Серверная часть (REST API)

### 8.1. Архитектура

Серверная часть построена на **FastAPI** с асинхронной обработкой запросов:

- **ORM**: SQLAlchemy 2.0 с Alembic миграциями
- **БД**: SQLite (с поддержкой PostgreSQL через Hyperdrive/другие)
- **Задачи**: asyncio + ThreadPoolExecutor для CPU-bound симуляций
- **Логирование**: loguru
- **Валидация**: Pydantic v2 с кросс-валидаторами

### 8.2. Эндпоинты API

| Группа | Путь | Методы | Назначение |
|:------:|:----:|:------:|:----------:|
| Health | `/api/v1/health` | GET | Статус сервера, версия, uptime |
| Upload | `/api/v1/upload` | POST, GET | Загрузка и парсинг FCS-файлов |
| Simulate | `/api/v1/simulate` | POST, GET, WS | Запуск, статус, отмена, WebSocket-прогресс |
| Analysis | `/api/v1/analysis` | POST, GET | Монте-Карло, чувствительность, оценка параметров |
| Results | `/api/v1/results` | GET | Получение траекторий симуляций |
| Export | `/api/v1/export` | POST | Экспорт результатов в CSV/JSON/PDF |
| Visualization | `/api/viz` | GET | Генерация графиков Plotly (JSON) |
| Spatial | `/api/v1/spatial` | GET | Снимки ABM-агентов и цитокиновых полей |

### 8.3. Безопасность и надёжность

В ходе аудита серверной части были выявлены и устранены **23 проблемы**:

**Критические (устранены):**
- Защита от Path Traversal при загрузке файлов (UUID-имена вместо клиентских)
- Валидация UUID во всех маршрутах (предотвращение инъекций в пути)
- Rollback сессий БД при исключениях
- Лимит размера загружаемых файлов (500 МБ, streaming-проверка)

**Высокий приоритет (устранены):**
- Ограничения skip/limit в пагинации (ge=0, le=500)
- Валидация статуса как enum (не произвольная строка)
- Кросс-валидация dt < t_max_hours
- Индексы в БД (status, created_at) для производительности
- Foreign Key между analyses и simulations

**Средний приоритет (устранены):**
- Таймаут симуляций (предотвращение бесконечных зависаний)
- Валидация WebSocket-подключений
- Атомарная запись файлов (temp → commit → move)
- Ограничение CORS (конкретные методы и заголовки)
- Graceful shutdown с отменой активных задач

### 8.4. Модели данных

3 основные таблицы с индексами:

```
uploads:      id (UUID PK), filename, status, metadata_json, created_at [+ indexes]
simulations:  id (UUID PK), mode, status, progress, params_json, result_path, created_at [+ indexes]
analyses:     id (UUID PK), analysis_type, status, simulation_id (FK), params_json, result_json [+ indexes]
```

## 9. Веб-интерфейс (Streamlit)

Фронтенд на **Streamlit** предоставляет интерактивный интерфейс:

**Боковая панель:**
- Начальные условия: n₀, c₀, доля стволовых клеток, макрофагов, уровень воспаления
- Время симуляции: t_max (5–90 дней), dt (0.001–0.05)
- Переключатели терапий: PRP (интенсивность), PEMF (частота, интенсивность)

**Вкладки:**
1. **Динамика** — графики популяций, цитокинов, ВКМ
2. **Фазы** — временная шкала фаз заживления
3. **Терапии** — кривые эффектов PRP/PEMF
4. **Ансамбль** — статистика Монте-Карло (среднее ± std, квантили)
5. **Настройки** — точная настройка 80+ параметров

## 13. Ключевые биологические механизмы

Платформа реализует следующие биологически обоснованные механизмы:

1. **Поляризация макрофагов M1 ↔ M2** — управляется IL-10 и TGF-β с обратным переключением через TNF-α
2. **Бистабильность TGF-β ↔ Mf** — положительная обратная связь, определяющая исход: нормальное заживление или фиброз
3. **Гипоксия-зависимый ангиогенез** — (1 − θ(O₂)) усиливает VEGF-ответ при низком кислороде
4. **Эффероцитоз** — поглощение нейтрофилов макрофагами → продукция IL-10 (противовоспалительный сдвиг)
5. **Бифазное высвобождение PRP** — быстрая фаза (α-гранулы, 0.5–2 ч) + медленная фаза (фибриновая сеть, 12–72 ч)
6. **Мультипутевой PEMF** — аденозиновые рецепторы + Ca²⁺-CaM/NO + MAPK/ERK
7. **Активация фибробласт → миофибробласт** — TGF-β-зависимая (путь Smad2/3)
8. **Баланс коллаген–MMP** — кинетика Михаэлиса-Ментен + ингибирование TIMP

## 14. Заключение и дальнейшие планы

### Текущее состояние

Платформа **RegenTwin** реализует полный цикл от загрузки экспериментальных данных до визуализации результатов мультимасштабного моделирования. Математическое ядро охватывает 20 биологических переменных, 80+ параметров, три масштаба моделирования и два типа терапевтических воздействий. Серверная часть прошла аудит безопасности и надёжности (23 исправления), тестовое покрытие составляет 85%.
